Java EE Apps

In 3 steps

  • Einfacher Prozess
  • 3 Schritte

#1: Write Code


@Stateless
public class MyService {

    @PersistenceContext
    private EntityManager entityManager;

    @TransactionalAttribute(MANDATORY)
    public void doStuff() {
        // business logic
    }

}
  • Anwendung entwickeln
  • Verwendung Java EE APIs
  • DI, Pooling, TX, Security
  • Wenige weitere Abh.

Containerless

Embedded Container

  • Buzzword: Containerless
  • -> Nicht korrekt
  • -> Einbetten des Containers
  • -> Umkehrung der Denkweise

Application

=

Code + Container

  • Applikation nicht nur Code
  • -> Zwei Bestandteile
  • Applikationscode + Runtime
  • Einbetten des Containers
  • Anwendung via main() starten
  • Full self contained

Why?

  • Lohnt der Aufwand?
  • Ja!
  • Viele Vorteile!

1 JVM = 1 App = 1 CL

  • Einfachere Welt!
  • EIN ClassLoader!
  • Statische Variablen! :)
  • Keine CL Leaks
  • CNF einfach zu debuggen
  • Kontrolle über Memory

Flexibility

  • Flexibilität
  • Komponenten: benötigt
  • Versionen: Neuste Patches
  • Updates einfacher
  • Kein Mehraufwand beim Deployment
  • Bsp: Wildfly 8.2

Simpler

Setup/Deployment

  • Anw. + Server = 1 Einheit
  • Kein separates Setup
  • Nur Konfiguration bleibt

Run/Debug

in your IDE

  • Schneller Einstieg
  • An Anwendung arbeiten
  • 1. In IDE importieren
  • 2. main() starten
  • Keine weiteren Anforderungen
